## Adapter contract (what every adapter returns)
Each adapter returns a **JS array** of raw grant objects (the pipeline validates them).
Provider-specific shaping lives inside the adapter:
- **Gemini** — REST `generateContent`, `response_mime_type: application/json` +
  `response_schema` = the array schema. Model text is a JSON array string → parse → array.
- **Anthropic** — Messages API tool-use. `input_schema` must be an object, so wrap the
  array schema as `{ type: object, properties: { grants: <schema> }, required: [grants] }`.
  Return `tool_use.input.grants` (already a parsed value — no `JSON.parse`).
- **Groq / OpenAI** — chat completions, `response_format: { type: "json_object" }`. JSON
  mode returns an object, so ask for `{"grants": [...]}` and return `parsed.grants`.

Model per provider is configurable via `GEMINI_MODEL` / `ANTHROPIC_MODEL` / `GROQ_MODEL` /
`OPENAI_MODEL`, each with a sensible default.

## Files
1. `scraper/src/providers/http.ts` — the injectable HTTP seam + shared request logic:
   - `FetchLike` / `HttpResponse` minimal interfaces (no DOM lib dependency).
   - `defaultFetch` wraps Node global `fetch`.
   - `postJson(fetchImpl, {url, headers, body, providerName})`: does the POST, classifies
     status → `ProviderError` (`429`/`5xx`/network = retryable; other non-2xx = not),
     returns the parsed envelope JSON.
   - `parseJsonText(text, providerName)`: `JSON.parse` or throw a non-retryable
     `ProviderError` ("invalid/truncated JSON").
2. `scraper/src/providers/retry.ts` — `withRetry(fn, opts)`: up to 3 attempts, exponential
   backoff (`base * 2^(n-1)`), only retries `ProviderError.retryable`, injectable `sleep`.
   `RetryOptions { retries?, baseDelayMs?, sleep? }`.
3. `scraper/src/providers/gemini.ts` — `GeminiProvider` (default). `ProviderConfig`.
4. `scraper/src/providers/anthropic.ts` — `AnthropicProvider`.
5. `scraper/src/providers/groq.ts` — `GroqProvider`.
6. `scraper/src/providers/openai.ts` — `OpenAIProvider`.
7. `scraper/src/providers/index.ts` — `getProvider(env = process.env) → LLMProvider`:
   reads `AI_PROVIDER` (default `gemini`), validates the matching key is present (clear
   Italian error if missing), throws on unknown provider.
8. `scraper/env.example` — add `GEMINI_API_KEY`, `ANTHROPIC_API_KEY`, `GROQ_API_KEY`,
   `OPENAI_API_KEY`, and the `*_MODEL` overrides (commented; only the active provider's key
   is required).

Shared `ProviderConfig { apiKey; model?; fetchImpl?; retry? }` lives in `http.ts`.
