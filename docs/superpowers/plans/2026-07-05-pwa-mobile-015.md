# Plan — branch 015 `feat/015-pwa-mobile`

> Depends on 006. Closes the desktop-first → mobile-friendly loop (ADR-006): a PWA manifest +
> icons (installable, standalone) and a systematic responsive pass for the fast-discovery
> mobile use case.

## Files
- `app/src/app/manifest.ts` — `MetadataRoute.Manifest`: name "BANDI-SCANNER", `lang: it`,
  `display: standalone`, `theme_color #2563eb`, 192/512 + maskable icons.
- `app/public/icons/*` — `icon-192/512`, `maskable-512`, `apple-touch-icon` (180), `icon.svg`,
  generated from an SVG with `sharp`.
- `app/src/app/layout.tsx` — real metadata (title/description/appleWebApp/icons) + a `viewport`
  export (`themeColor`, `width=device-width`), `lang="it"`.
- `app/src/app/(app)/layout.tsx` — nav gets classes (`app-nav`/`app-tabs`/`app-brand`) so it can
  become a **bottom tab bar under 760px**.
- `app/src/app/globals.css` — responsive refinement: centered `main` (max-width), no page-level
  horizontal overflow, images `max-width:100%`, the bottom tab bar, **44px tap targets** and
  full-width form controls on touch layouts.

## Tests / verification
- `manifest.test.ts` — standalone/it/name/theme + 192/512/maskable icons present.
- Browser check (Playwright, this branch): `/login` and `/signup` at **375px** and `/login` at
  **1280px** → **no horizontal overflow** (scrollWidth == viewport). Screenshot at 375px reviewed.
- Manual (post-deploy, documented): the 4 app pages at 375/768/1280 without overflow or
  sub-44px targets; Lighthouse PWA installable + a11y ≥ 90; "Add to Home Screen" on real
  Android/iOS opening standalone.

## Notes

installability, not a visual redesign.
