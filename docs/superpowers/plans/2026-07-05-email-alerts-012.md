# Plan — branch 012 `feat/012-email-alerts`

> Depends on 006. Weekly email digest (§5.4): the week's new grants above the user's score
> threshold (default 50), via Resend, preferences in `user_settings`, a dedicated cron.

## Design
- **`build-digest.ts`** (pure) — `buildDigest(profile, threshold, newViews)`: reuses
  `buildMatchedGrants`, keeps score ≥ threshold (excludes "Storico"), sorts desc, caps 10;
  `null` when empty (no empty emails). The 7-day "new" scoping is the caller's query.
- **`render-email.ts`** — Italian HTML with inline styles (email-client safe), compact cards
  (title, provider, score, verdict, deadline, link to the detail), footer "Gestisci le
  preferenze" → `/profilo`. HTML-escaped.
- **`send.ts`** — `EmailSender` seam; `ResendSender` (POST api.resend.com/emails, injectable
  fetch) + `FakeEmailSender` for tests; `getSender(env)` needs `RESEND_API_KEY`.
- **`run-batch.ts`** — `runDigestBatch(deps)` with all I/O injected: fetch the week's grants
  once, then per recipient build+render+send; a per-user failure is logged and counted, never
  fatal to the batch.
- **`api/cron/digest/route.ts`** — CRON_SECRET-protected; wires the real deps over the
  service-role admin client (weekly `user_settings`, profile, `auth.admin.getUserById` for the
  email, new grants). `maxDuration 300`.
- **`lib/settings/actions.ts`** `updateAlertSettings(threshold, frequency)` (upsert), profile
  **"Notifiche"** section (`section-notifications.tsx`: frequency + 0–100 threshold slider).
- **`vercel.json`** — `+ { path: /api/cron/digest, schedule: "0 7 * * 1" }` (Mon 07:00).

## Env (set in Vercel; documented, not committed)
`RESEND_API_KEY`, optional `DIGEST_FROM_EMAIL` (verified sender), `NEXT_PUBLIC_APP_URL` (for
absolute links in the email).

## Tests (offline)
- `build-digest.test.ts` — threshold respected, raised threshold, cap at 10 sorted desc,
  empty → null.
- `run-batch.test.ts` (fake sender) — sends to a qualifying recipient; below-threshold →
  no send; empty recipients → nothing; **one recipient throwing → the others still receive**.
  Plus `renderDigestEmail` subject/links/escaping.
- `digest-route.test.ts` — 401 without / with a wrong secret.

## Acceptance (roadmap)
- Manual cron trigger on staging → email received & rendered (Gmail), links work → manual
  post-deploy (needs `RESEND_API_KEY`). The batch/threshold/empty logic is unit-tested here.
- Below threshold / alerts off / no new grants → nothing sent (tested).
- Changing the threshold in the profile changes the next digest (settings action + build-digest).
