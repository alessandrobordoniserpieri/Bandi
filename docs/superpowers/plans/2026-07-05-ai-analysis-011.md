# Plan — branch 011 `feat/011-ai-analysis`

> Depends on 005 (grant detail) + 008 (real LLM adapters). Two on-demand AI features reusing
> the `LLMProvider` seam: (a) **Analisi AI approfondita** on the grant detail — a
> profile×grant analysis with the GDPR/AI disclaimer (§8); (b) **Segnala un bando** (§4.4)
> crowdsourcing — the user pastes a URL, the AI structures it into the 16 fields, the grant
> becomes available to everyone.

## Source of truth
- Roadmap `#b011`; §8 (disclaimer), §4.4 (crowdsourcing).
- ADR-008: the app consumes the scraper via its public entrypoint only.
- RLS: `grants` is **select-only** for authenticated → submit-url inserts go through the
  **admin (service-role) client**, server-side only.

## Design decisions
1. **Scraper exports, not new code**: `index.ts` additionally exports `getProvider`,
   `extractGrants`, `enrich`, `normalizeUrl`, `BrowserlessFetcher`, `SupabaseGrantsDb`,
   `FakeLLMProvider`, `ProviderError` + the seam types. The roadmap's `lib/ai/provider.ts` is
   a thin re-export of `getProvider` (the only AI-provider touchpoint in the app).
2. **Analysis output is never raw**: `analyzeGrant` asks the provider for a JSON object
   `{punti_di_forza[], rischi[], suggerimenti[], passi_successivi[]}` and validates with zod
   (strings trimmed, arrays non-empty overall). Malformed output → Italian error, nothing
   AI-generated is rendered unvalidated.
3. **Rate limit persisted in `user_settings`** (roadmap: "in user_settings o header"):
   migration 0007 adds `ai_calls_count int` + `ai_calls_window_start timestamptz`. Route logic:
   upsert row; if the window (1h) expired → reset; if count ≥ 10 → 429; else increment.
   Uses the user's client (owner RLS applies). Races are tolerable for an MVP limit.
4. **Submit-url is two-phase** (roadmap wants "anteprima + conferma"): `preview` fetches via
   Browserless + extracts (LLM) and returns the fields **without inserting**; `confirm`
   receives the previewed grant back, **re-validates it with a strict zod schema against the
   app's own vocab** (`LEGAL_TYPES`/`TAGS`/`DOCUMENT_KEYS` — tampering can't produce values
   the scraper itself wouldn't), dedups by normalized URL, inserts with `source_id = null`,
   `import_mode = 'user'` via the admin client. Avoids paying the LLM twice.
5. Migration 0007 also adds `grants.import_mode text not null default 'scraper'`
   (check: `scraper|user`) so crowdsourced grants are distinguishable.

## Files
- `scraper/src/index.ts` — extra exports (no behavior change).
- `app/supabase/migrations/0007_ai_analysis.sql` — import_mode + rate-limit columns.
- `app/src/lib/ai/provider.ts` — re-export `getProvider`.
- `app/src/lib/ai/analyze-grant.ts` — prompt builder (profile + 16 grant fields, Italian) +

- `app/src/app/api/ai/analyze/route.ts` — POST `{grantId}`: auth (401) → rate limit (429) →
  load profile+grant (404) → analyze (502 on provider/validation failure) → JSON. `maxDuration 60`.
- `app/src/components/grants/ai-analysis-panel.tsx` — client: button → loading → panel with
  the disclaimer «Analisi generata da AI: verifica sempre il testo ufficiale del bando».

  injected deps (fetcher, llm, adminDb) so tests run offline.
- `app/src/app/api/grants/submit-url/route.ts` — POST `{action, url, grant?}`, auth, `maxDuration 60`.
- `app/src/components/grants/submit-url-dialog.tsx` — Dashboard: URL input → preview → confirm.
- `app/(app)/page.tsx` + `bandi/[id]/page.tsx` — wire the two components.
- `database.types.ts` — patch for the new columns.

## Tests (offline)
- `analyze-grant.test.ts` — valid provider output → parsed structure; malformed/missing keys →
  throws (route maps to Italian error); prompt contains real profile + grant elements.
- `submit-url.test.ts` — URL already present → `exists`, no insert; empty extraction →
  `not_a_grant`, no insert; happy path → insert row with `import_mode: 'user'`, `source_id: null`,
  normalized URL; confirm payload failing vocab validation → error, no insert.
- Route tests — both routes return 401 unauthenticated; analyze returns 429 over the limit.

## Acceptance (roadmap)
- Analysis in Italian, <30s with Gemini, disclaimer shown, cites real profile elements —
  prompt embeds the profile; latency is a runtime property (needs a real key).
- A real pasted grant URL produces a grant visible to another user — insert path verified
  offline; real-run needs Browserless + key (documented in PR).
- Unauthenticated → 401 on both routes (tested).
